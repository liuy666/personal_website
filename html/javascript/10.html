事件驱动机制：

1、事件的概念：

JavaScript使我们有能力创建动态页面，网页中的每一个元素都可以产生某些触发JavaScript函数的事件。

我们可以认为事件是可以被JavaScript侦测到的一种行为。

2、事件类型：

click、mousemove、keydown...... 事件源 : DOM元素、window对象

事件监听器 :事件处理程序（事件响应程序：函数）

Event对象： 代表事件的状态，比如事件在其中发生的元素、键盘按键的状态、鼠标的位置、鼠标按钮的状态。

对象获取：
    标准：事件处理程序的第一个参数
    非标准(IE9以前)：window.event

    兼容写法：
        event = event || window.event;

鼠标属性：(包含滚动条部分)
    event.clientX -- 当前可见视口中x坐标
    event.clientY -- 当前可见视口中y坐标
    event.pageX -- 相对文档来说的x坐标
    event.pageY -- 相对文档来说的y坐标
    event.pageX || (event.clientX + (document.documentElement.scrollLeft || document.body.scrollLeft))
    event.pageY || (event.clientY + (document.documentElement.scrollTop || document.body.scrollTop))
    event.offsetX -- 事件源元素坐标系中x坐标
    event.offsetY -- 事件源元素坐标系中y坐标
    event.button -- 鼠标键
    event.which -- 鼠标键

键盘属性：
    event.which -- 获取按键的虚拟键盘码
    event.keyCode -- 获取按键的虚拟键盘码
    event.which || event.keyCode
    keydown/keyup 事件中获取到的是虚拟键盘码
    keypress 事件中获取到的是字符编码
    event.ctrlKey
    event.altKey
    event.shiftKey

事件源属性：
    event.target -- 获取最初触发事件的事件源元素
    event.srcElement
    event.currentTarget -- 

获取窗口滚动距离：
    document.documentElement.scrollTop || document.body.scrollTop
    document.documentElement.scrollLeft || document.body.scrollLeft

事件冒泡：

从最初触发事件的事件源元素开始，沿着DOM树向自己的祖先元素传播事件的行为。

集中式事件处理

事件委派（代理）：

利用事件冒泡的特性，将后代元素上事件的处理委派给祖先元素。事件委派过程中，可以通过 event.target 获取最初触发事件的事件源元素。

事件委派适用于未来元素（动态添加的元素）

现代浏览器中事件的处理默认都是按照事件冒泡的方式来处理事件。

事件冒泡可能会带来副作用，如果希望事件停止传播，则需要阻止事件冒泡：

标准：event.stopPropagation();
非标准(IE9之前)：event.cancelBubble = true;

事件流：

a. 事件捕获

从DOM树结构上层开始向后代元素捕获处理事件的流程
从最不特定元素开始到最特定元素结束的事件处理流程

b. 处于目标

c. 事件冒泡

添加事件监听的方法：

element.addEventListener(type, callback[, useCapture])

type -- 事件类型（字符串），如 click、mousemove...
callback -- 事件处理程序（回调函数）
useCapture -- 是否使用事件捕获的方式来处理事件，可选参数。默认为 false
        -- true 表示使用事件捕获，false 不使用（事件冒泡）
        -- 如果为 true 表示元素绑定的事件是在捕获阶段处理
        -- 如果为 false 表示元素绑定的事件是在冒泡阶段处理

IE9之前：

element.attachEvent(type, callback);
type -- 事件类型，需要添加 on 前缀
callback -- 事件处理程序

移除事件监听：

removeEventListener(type, callback[, useCapture])
detachEvent(type, callback)  是否也要加on？

注册事件监听：

1. addEventListener()   ---- DOM2，推荐
2. 事件句柄             ---- DOM0
3. HTML标签事件属性       ---- DOM0

this 的使用：

a. 构造函数中的this通常指向通过new所创建的对象本身
b. 事件处理程序（函数）中的 this 通常指向的是绑定事件的事件源元素
